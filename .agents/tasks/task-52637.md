# LT-52637: Перевести поведение гибридных субагентов с ID Мосгортур на флаг партнёра

## 1. Бизнес-контекст для задачи

В приложении есть партнёр Мосгортур (`Partner::MOSGORTUR_ID`), для которого накопилось несколько специальных веток поведения. Планируется подключить ещё одного партнёра с таким же поведением гибридного субагента. Расширять каждую проверку списком ID (`partner.id.in?([Partner::MOSGORTUR_ID, Partner::ANOTHER_ID])`) нельзя: это продолжит распространять хардкод и потребует изменений кода при подключении каждого следующего партнёра.

Нужно добавить в таблицу `partners` boolean-поле `is_subagent_hybrid`, которое описывает бизнес-возможность партнёра. По умолчанию поле выключено. Для Мосгортур и будущих гибридных субагентов флаг будет включаться вручную через ActiveAdmin после выкладки.

Проверки `Partner::MOSGORTUR_ID`, относящиеся именно к общему поведению гибридных субагентов, нужно заменить на проверку флага. Проверки, которые выбирают конкретные интеграции, credentials, бренд или URL Мосгортур, должны остаться привязанными к `Partner::MOSGORTUR_ID`.

Целевой результат:

- новое гибридное поведение подключается настройкой партнёра без добавления его ID в прикладной код;
- существующее поведение Мосгортур сохраняется после ручного включения флага;
- специфичные интеграции Мосгортур не распространяются на других гибридных партнёров;
- флаг виден и редактируется в ActiveAdmin под названием «Гибридный субагент».

## 2. Ключевые решения и обоснования

1. **Добавить `partners.is_subagent_hybrid` как `boolean`, `default: false`, `null: false`.** Поле имеет однозначную бизнес-семантику и не должно иметь третьего состояния `NULL`.

2. **Не выполнять автоматический backfill для Мосгортур в миграции.** Пользователь подтвердил, что флаг будет включён вручную. После выкладки кода необходимо сразу включить его у партнёра `Partner::MOSGORTUR_ID`, иначе обобщённые ветки поведения временно перестанут применяться к Мосгортур.

3. **Добавить флаг во все необходимые поверхности `Partner`.** Поле нужно разрешить через существующий `attr_accessible`, вывести в show и form ActiveAdmin и добавить перевод атрибута `activerecord.attributes.partner.is_subagent_hybrid: Гибридный субагент` в `config/locales/ru.yml`.

4. **Не удалять `Partner::MOSGORTUR_ID`.** Константа остаётся необходимой для сертификатного API, SMS-аккаунта, ссылки личного кабинета, бренда отправителя писем и других конкретных интеграций Мосгортур.

5. **Не наследовать гибридного партнёра из предыдущего заказа.** В [`Order::AssignPartnerData`](../../app/services/order/assign_partner_data.rb#L96) проверку `Partner::MOSGORTUR_ID` заменить на `previous_order.partner&.is_subagent_hybrid?`. Отдельные условия для `is_outer_first_line` и `Order::TOURVISOR_ID` сохранить.

6. **Не создавать брошенные корзины для гибридных партнёров.** В [`AbandonedCartsCache`](../../app/services/abandoned_carts_cache.rb#L5) оставить ID-исключение для `Partner::ALFA_WL_ID`, а в [`Order#write_to_abandoned_carts_cache?`](../../app/models/order.rb#L3543) добавить проверку `partner&.is_subagent_hybrid?`. Не помещать динамическое множество партнёров обратно в константу ID.

7. **Не отправлять внутреннее premium-уведомление по гибридным заказам.** В [`Order#premium_with_notification?`](../../app/models/order.rb#L3553) заменить только исключение Мосгортур на проверку флага через партнёра; исключения `ALFA_WL_ID`, `OZON_IDS` и `ALFA_BANK_ID` сохранить.

8. **Блокировать CRM-автоответы для всех гибридных партнёров.** В [`CrmLevel::Email`](../../lib/engines/crm_level/app/models/crm_level/email.rb#L185) проверки партнёра заказа и последнего заказа клиента перевести на `partners.is_subagent_hybrid`, сохранив эффективные SQL-запросы без N+1. Методы с `mosgortur` в названии переименовать по бизнес-смыслу. Проверку конкретной почтовой группы [`CrmLevel::MailGroup::MOSGORTUR_ID`](../../lib/engines/crm_level/app/models/crm_level/email.rb#L225) оставить специфичной для Мосгортур.

9. **Добавлять CRM-тег для всех гибридных партнёров.** В [`CrmLevel::AddPartnerTagsToTicket`](../../lib/engines/crm_level/app/services/crm_level/add_partner_tags_to_ticket.rb#L3) сохранить явные исключения `Partner::ALFA_WL_ID` и `Partner::TUTU_ID`, а Мосгортур и будущих гибридных партнёров выбирать по флагу. Условия для заказов тикета и последнего заказа клиента должны быть согласованы.

10. **Использовать `Operator::VERSAL_ID` как fallback для всех гибридных партнёров.** В [`PartnerHooksHandlers::PackageBuilder`](../../app/services/partner_hooks_handlers/package_builder.rb#L66) заменить `mosgortur_partner?` на предикат с бизнес-названием и проверкой `partner&.is_subagent_hybrid?`.

11. **Разрешать recurrent refund всем гибридным партнёрам.** В [`Order#reccurent_refund_allowed?`](../../app/models/order.rb#L3582) заменить сравнение `partner_id` с ID на `partner&.is_subagent_hybrid?`. Существующее имя метода вне рамок задачи не переименовывать, чтобы не расширять diff без необходимости.

12. **Запрещать fallback на следующий эквайринг всем гибридным партнёрам.** В [`PaymentRouter#fallback_to_next_acquiring_allowed?`](../../app/apis/payment_router.rb#L77) использовать отрицание `payment.order&.partner&.is_subagent_hybrid?`, сохранив разрешённый fallback для заказа без партнёра.

13. **Выдавать `supplier_order_data` всем гибридным партнёрам.** В [PAPI v3 order partial](../../app/views/papi/v3/clients/_order.json.jbuilder#L255) заменить ID на `@partner.is_subagent_hybrid?`, сохранив альтернативное условие `params[:supplier_info] == 'true'`. Поскольку меняется условие выдачи части PAPI v3-контракта, синхронно обновить документацию по `.agents/docs/papi_v3_docs.md`.

14. **Оставить SMS-логику специфичной для Мосгортур.** Проверки в [`SMS::Checker`](../../app/base/sms/checker.rb#L28) и registry в [`SMS::Account`](../../app/base/sms/account.rb#L34) выбирают конкретные sender и credentials. Флаг гибридности не содержит данных для выбора SMS-аккаунта.

15. **Оставить redirect в личный кабинет специфичным для Мосгортур.** [`OrdersController#partner_redirect`](../../app/controllers/orders_controller.rb#L540) и [`Partner::LK_LINKS`](../../app/models/partner.rb#L39) используют конкретный домен Мосгортур.

16. **Оставить сертификатную интеграцию специфичной для Мосгортур.** Выбор клиента в [`PartnerCertificates::Validator`](../../app/services/partner_certificates/validator.rb#L23), токен и partner ID логов в [`PartnerCertificates::Clients::Mosgortur`](../../app/services/partner_certificates/clients/mosgortur.rb#L13) относятся к API Мосгортур. По той же причине сертификатный блок в [PAPI v3 order partial](../../app/views/papi/v3/clients/_order.json.jbuilder#L48) остаётся по ID.

17. **Оставить бренд отправителя письма специфичным для Мосгортур.** [`OrderHotelReviewMailer#sender_from`](../../app/mailers/order_hotel_review_mailer.rb#L72) возвращает буквальное имя `"МОСГОРТУР"`; включение флага у другого партнёра не должно приводить к отправке писем от чужого бренда.

18. **Исправить вводящий в заблуждение сценарий `Notifier` только в тесте.** В production-коде [`Notifier#mute_notifications?`](../../app/services/notifier.rb#L97) отсутствует проверка `Partner::MOSGORTUR_ID`: SMS отключается общим полем `partner.disable_transactional_sms?` при выполнении условий `transactional_sms?` и `order.from_wl?`. Мосгортур фигурирует только в отдельном контексте [`notifier_spec.rb`](../../spec/services/notifier_spec.rb#L187), где его ID используется как fixture, хотя не влияет на проверяемую ветку. Контекст нужно переименовать по реальному поведению и использовать обычного партнёра без фиксированного ID.

19. **Тестировать обобщённое поведение на произвольном партнёре с включённым флагом.** Например, сценарии fallback-оператора в [`order_creator_spec.rb`](../../spec/services/partner_hooks_handlers/tourvisor/order_creator_spec.rb#L240) и CRM-тегов в [`add_partner_tags_to_ticket_spec.rb`](../../lib/engines/crm_level/spec/services/add_partner_tags_to_ticket_spec.rb#L7) не должны доказывать гибридность через `Partner::MOSGORTUR_ID`. Специфичные SMS-, redirect- и certificate-тесты остаются на ID Мосгортур.

### Критически важные принятые решения

- Новый флаг называется строго `is_subagent_hybrid`.
- Флаг является единственным расширяемым критерием общего поведения гибридных субагентов; списки новых partner ID для этих веток не добавляются.
- Миграция не изменяет данные существующих партнёров. Мосгортур должен быть включён вручную сразу после выкладки.
- `Partner::MOSGORTUR_ID` не удаляется и остаётся в специфичных интеграциях.
- По флагу обобщаются: наследование партнёра предыдущего заказа, брошенные корзины, premium-уведомления, CRM-автоответы, CRM-теги, fallback на Versal, recurrent refund, запрет fallback эквайринга и выдача `supplier_order_data`.
- По ID Мосгортур остаются: SMS account/checker, redirect и `LK_LINKS`, сертификатный API/логи/PAPI-блок, бренд отправителя письма и конкретная CRM mail group.
- `Notifier` в production не содержит отдельного MGT-условия: тестовый MGT-контекст должен быть нейтрализован, потому что проверяемое поведение основано на `disable_transactional_sms`.
- Изменение условия `supplier_order_data` является изменением PAPI v3-контракта и требует документации в том же PR.
- Любые проверки флага в выборках CRM должны быть реализованы без загрузки коллекций и без N+1.
